Module contents¶

The input package contains all input modules for the EEG-Kiss framework. These modules contain both offline as online data sources. Offline data sources include filereaders such that data can be loaded from the hard disk. Online data sources are streams of data, for instance from an EEG headset or online sources.

root.spikes.data_simulator_plot.oscdatamain module¶
root.spikes.data_simulator_plot.oscdatamain.simulate()[source]¶

This function starts four separate processes:

1: A thread playing back two channels of data from one EEG file

2: A thread playing back the same two channels from a coupled EEG file (These two files correspond to two kissing people)

3: A thread is plotting one channel’s data of the people and forwarding the other channel’s data to another port

4: A thread is plotting the second channel’s data, forwarded by the third thread

root.spikes.bandpassfilterspike module¶

This python program opens a predefined HDF file then filters the data and visualizes the data EEG.

The HDF file contains six types of data for eight channels. The EEG stream for the first channel is visualised.

Used libraries are:

mne -> for filtering and other EEG analytics

numpy -> for mathmetics

MatplotLib -> for plotting

h5py -> for reading HDF files

root.spikes.bandpassfilterspike.plotLotsOfData()[source]¶

This function is basically a script for trying out a number of functionalities.

1: First, a file is read using the HDFFilreader.

2: From that file, two EEG channels are isolated.

3: Different band-pass filters are applied using MNE’s Filter package

4: Activation of a filter band in a time-window is calculated (!!! But this is done in the wrong way! Do not use this!!!)

5: The left and right channels filtered information is combined into valence and arousel values

6: All these steps are repeated for the paired data file

7: All this information is plotted
